Frontend (Vercel)

  • Desarrollado con HTML, CSS (Tailwind) y JavaScript/jQuery
  • Interfaz responsiva y moderna
  • Comunicación con API mediante AJAX
  • Alojado en Vercel para máximo rendimiento

Backend (.NET en Somee)

  • Desarrollado con .NET Core / .NET 6+
  • Arquitectura en capas (Controladores, Servicios, Repositorios)
  • API RESTful con endpoints para acortar URLs y obtener analytics
  • Alojado en Somee con base de datos SQL Server

Flujo de Datos para Acortar una URL

sequenceDiagram participant U as Usuario participant F as Frontend (Vercel) participant A as API .NET (Somee) participant S as Servicio participant R as Repositorio participant DB as SQL Server (Somee) U->>F: Ingresa URL larga y hace clic en "Acortar" F->>A: POST /api/urls (con URL original) A->>S: ProcessUrl(url) S->>S: Generar código único S->>R: GuardarUrl(urlOriginal, codigo) R->>DB: INSERT en tabla URLs DB-->>R: ID generado R-->>S: Entidad URL guardada S-->>A: URL acortada (response DTO) A-->>F: Respuesta JSON con URL acortada F->>F: Mostrar URL acortada con indicador de éxito F-->>U: Mostrar resultado al usuario

Endpoints Principales de la API

Método Endpoint Descripción Body/Params
POST /api/urls Acortar una nueva URL { "originalUrl": "https://..." }
GET /api/urls/{codigo} Redireccionar a URL original Código en URL
GET /api/urls/{codigo}/analytics Obtener analytics de URL Código en URL
GET /api/urls/{codigo}/clicks Obtener datos de clics Código en URL
